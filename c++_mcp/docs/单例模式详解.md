# 单例模式详解

## 概述

本项目使用了 **饿汉式 + 线程安全** 的单例模式实现，共有两个单例：
1. `logger` - 日志记录器
2. `resource_manager` - 资源管理器

---

## 1. logger 单例 (饿汉式 - 推荐)

### 代码实现

```cpp
// mcp_logger.h

class logger {
public:
    // 最简洁的线程安全单例
    static logger& instance() {
        static logger instance;  // C++11 保证线程安全
        return instance;
    }
    
    void set_level(log_level level);
    template<typename... Args> void debug(Args&&... args);
    template<typename... Args> void info(Args&&... args);
    template<typename... Args> void warning(Args&&... args);
    template<typename... Args> void error(Args&&... args);

private:
    logger() : level_(log_level::info) {}
    
    log_level level_;
    std::mutex mutex_;
};
```

### 为什么用 static 局部变量？

```cpp
// C++11 及之后，以下写法是线程安全的：
static logger& instance() {
    static logger instance;  // 只有第一次调用时初始化
    return instance;
}
```

**原理**：
- C++11 标准规定：局部 static 变量的初始化是线程安全的
- 编译器会自动添加锁来保证只有一个线程执行初始化
- 之后的调用直接返回已初始化的对象

### 优点

1. **简单**：一行代码搞定
2. **线程安全**：C++11 标准保证
3. **延迟初始化**：第一次使用时才创建，节省资源
4. **自动清理**：程序结束时自动销毁

## 2. resource_manager 单例

### 代码实现

```cpp
// mcp_resource.h (头文件)
class resource_manager {
public:
    static resource_manager& instance();
    // ... 公共接口

private:
    resource_manager() = default;  // 私有构造函数
    ~resource_manager() = default;
    
    // 禁用拷贝和赋值
    resource_manager(const resource_manager&) = delete;
    resource_manager& operator=(const resource_manager&) = delete;
    
    std::map<std::string, std::shared_ptr<resource>> resources_;
    std::map<int, std::pair<std::string, std::function<void(const std::string&)>>> subscriptions_;
    int next_subscription_id_ = 1;
};

// mcp_resource.cpp (实现文件)
static std::mutex g_resource_manager_mutex;

resource_manager& resource_manager::instance() {
    static resource_manager instance;  // 线程安全
    return instance;
}
```

### 私有构造函数的作用

```cpp
private:
    resource_manager() = default;  // 可以初始化成员
    
    // 禁止拷贝构造
    resource_manager(const resource_manager&) = delete;
    
    // 禁止赋值
    resource_manager& operator=(const resource_manager&) = delete;
```

**防止的问题**：
```cpp
// 如果不禁用，编译器可能会生成这些：
resource_manager a;           // 普通构造
resource_manager b = a;       // 拷贝构造 ❌
resource_manager c;           
c = a;                        // 赋值 ❌

// 这会导致多个实例存在，违背单例原则
```

### 双重保护机制

```cpp
// 除了 static 局部变量，还有 mutex 保护数据
static std::mutex g_resource_manager_mutex;

void register_resource(std::shared_ptr<resource> resource) {
    std::lock_guard<std::mutex> lock(g_resource_manager_mutex);
    resources_[uri] = resource;  // 操作资源时加锁
}
```

## 3. 两种单例模式对比

| 特性 | 懒汉式 | 饿汉式 (本项目) |
|------|--------|----------------|
| 初始化时机 | 第一次使用时 | 程序启动时 |
| 线程安全 | 需要双重检查锁定 | C++11 自动保证 |
| 性能 | 首次调用有开销 | 始终最优 |
| 资源占用 | 延迟释放内存 | 程序结束才释放 |

### 本项目使用的模式

```cpp
// 本质上是 "懒汉式 + C++11 线程安全局部变量"
static resource_manager& instance() {
    static resource_manager instance;  // 第一次调用时才创建
    return instance;
}
```

## 4. 使用示例

### logger 单例

```cpp
// 通过宏直接使用
LOG_INFO("Server started on port ", 8080);
LOG_ERROR("Connection failed: ", e.what());

// 或直接调用
mcp::logger::instance().info("Message");
```

**宏定义**：
```cpp
#define LOG_INFO(...) mcp::logger::instance().info(__VA_ARGS__)
```

### resource_manager 单例

```cpp
// 获取单例
auto& rm = resource_manager::instance();

// 注册资源
rm.register_resource(my_resource);

// 获取资源
auto resource = rm.get_resource("file://config.json");

// 订阅资源变化
rm.subscribe("file://config.json", [](const std::string& uri) {
    std::cout << "Resource changed: " << uri << std::endl;
});
```

## 5. 单例模式在项目中的意义

### logger 的作用
- **全局日志**：整个程序的日志输出统一管理
- **线程安全**：多线程同时写日志不会混乱
- **统一格式**：统一的日志格式和时间戳

### resource_manager 的作用
- **全局资源池**：所有注册的资源统一管理
- **订阅机制**：支持多个观察者订阅同一资源
- **避免重复**：防止创建多个管理器导致状态不一致

## 6. 单例模式的常见陷阱

### 1. 线程安全 (旧式 C++)

```cpp
// ❌ 错误：C++11 之前不是线程安全的
class BadSingleton {
    static BadSingleton* instance;
    static BadSingleton* getInstance() {
        if (instance == nullptr) {  // 问题：可能多个线程同时进入
            instance = new BadSingleton();
        }
        return instance;
    }
};

// ✅ 正确：双重检查锁定 (C++11 之前)
class ThreadSafeSingleton {
    static std::atomic<ThreadSafeSingleton*> instance{nullptr};
    static std::mutex m;
    
    static ThreadSafeSingleton* getInstance() {
        if (instance == nullptr) {
            std::lock_guard<std::mutex> lock(m);
            if (instance == nullptr) {
                instance = new ThreadSafeSingleton();
            }
        }
        return instance;
    }
};
```

### 2. 析构函数顺序问题

```cpp
// 如果单例A依赖单例B，B必须比A先销毁
// C++ 不保证静态对象的销毁顺序，可能导致问题

// 解决方案：使用 static 局部变量（按需创建，自动销毁）
```

## 7. 总结

本项目单例模式的核心实现：

```cpp
// 简洁、安全、推荐
static T& instance() {
    static T instance;
    return instance;
}
```

**优点**：
1. 线程安全（C++11+）
2. 延迟初始化
3. 简单易维护
4. 自动资源管理
